## v1.4.1 — pixel-exact board at every zoom, every device

**Fixed the empty band at the bottom of the board frame** that appeared on desktop after zoom/resize and could appear on mobile in some viewport configurations.

### Root cause

v1.4.0 set `--board-h` from JS and derived `--board-w` via a CSS `calc(--board-h / 2)`, with a 200 ms transition on the property. After a zoom or resize, the renderer (called after a double-rAF) measured the canvas mid-tween: width was still catching up while the height kept growing toward its final value. The renderer locked in `cell = rect.width / 10` for an intermediate width, so the play grid stopped short of the bottom of the frame.

### The fix

`cell` (in integer CSS pixels) is now the single source of truth.

```
cellFit = floor( min( availH / 20, availW / 10 ) )
cell    = clamp( CELL_MIN, floor(cellFit * zoom), CELL_MAX )

--board-h = cell * 20
--board-w = cell * 10
```

CSS no longer derives one dim from the other, and there's no transition on the board props — the snap is instant, so `getBoundingClientRect()` and `canvas.width` never disagree.

### What changed

- `app.js` — new `applyBoardSize()` computes integer cell, sets both `--board-h` and `--board-w` explicitly.
- `game.css` — `.game-wrap` no longer derives `--board-w` from `--board-h`; transition on `--board-h` removed; `@property --board-h` decl removed (no longer needed).
- Service worker cache bumped to `tetris-v1.4.1`.
