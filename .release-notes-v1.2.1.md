# tetris v1.2.1 — polish pass

A small but satisfying patch release on top of [v1.2.0](https://github.com/paulfxyz/tetris/releases/tag/v1.2.0). Everything you noticed in the last play session is now fixed.

## ✨ Added

- **Hold-box discoverability** — the `HOLD` panel now shows `C or Shift` on desktop and `tap H` on touch devices, so new players actually find the swap mechanic.
- **Smart brand bar** — clicking the `tetris` logo reloads the page; the `v1.2.1` badge next to it links to this exact GitHub release in a new tab.

## 🩹 Fixed

- **Canvas trail on zoom-out.** When the board was scaled down via CSS, `drawBoard()` cleared in CSS-rect coordinates while the canvas bitmap kept its larger dimensions — leaving stale pixels visible as diagonal smear trails below the bottom row. The board now wipes the **full bitmap** every frame (same robust pattern that `drawMini` was already using for the HOLD / NEXT panels), and manual zoom triggers `renderer.resize()` so the bitmap matches the new CSS size.
- **Zoom +/− buttons now persist.** Clicking "Fit to screen" had set an internal `fit` flag that `applySettings()` honored on every subsequent action, immediately overwriting any manual zoom. Manual `+` / `−` now clears that flag so your choice sticks.
- **Skip button in the score modal closes the dialog.** It was silently swallowed by HTML5 form validation on the `required` Name input. Adding `formnovalidate` lets Skip bypass validation and close cleanly.

## 🎨 Changed

- Score-save modal status line is friendlier and clearer:
  > Your score will be PGP signed upon download or submission. — Server reachable.
- Modal placeholders refreshed: `John Doe` (name), `I'm a Tetris fan` (tagline).
- Service-worker cache bumped to `tetris-v1.2.4` to force-refresh existing players.
