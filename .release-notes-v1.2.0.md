## v1.2.0 — real music + actually-signed scoreboard receipts

Two long-standing issues finally fixed.

### 🎵 Real music

For two versions the music toggle did nothing audible for most players. The procedural Korobeiniki Web Audio synth from v1.1.x looked great on paper but failed silently in the wild — autoplay restrictions, tab throttling, and AudioContext lifecycle edge cases all conspired to make the lookahead scheduler emit no sound.

v1.2.0 ships a real bundled MP3:

- **"Happy Adventure (Loop)"** by Bart Kelsey ([opengameart.org](https://opengameart.org/content/happy-adventure-loop)) — CC0 / public domain, ~620 KB, 8-bit / chiptune vibe, loops seamlessly.
- Played via a plain `<audio loop>` element. No Web Audio scheduling, no oscillators, no race conditions.
- Service worker precaches it too — works offline.

### 🔐 Signed receipts now actually sign

`/api/health` was reporting `key:false` on production. Root cause: PHP resolved the private-key directory to `<webroot>/../private/`, which on Siteground's per-domain layout is `~/tetris.rocks/private/` — a different directory than the FTP-account-root `~/private/` where the keys lived. Empty directory, no key, no signature.

Two fixes:

1. **PHP probes multiple candidate locations** for the private key (per-domain, account-root, `$HOME`) and uses whichever actually contains `pgp-private.asc`.
2. **The deploy script uploads keys to both locations** as belt-and-suspenders.

Downloaded `.txt` receipts are now real PGP cleartext-signed messages, verifiable with `gpg --verify <file>` against the public key at https://tetris.rocks/pubkey.asc (fingerprint `70B07D44B5D29BABAE757108E422C976C88EA51B`).

`/api/health` also now reports `private_dir` and `priv_key_present`, so future key-misplacement issues are visible at a glance.

### Other
