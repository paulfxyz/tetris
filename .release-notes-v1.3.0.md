## 🎯 Pixel-perfect zoom on every device

Zooming the board in and out used to leave either an **empty strip below the last row** or a **clipped bottom row**, depending on the viewport. Both bugs had the same root cause: `--board-h` could land on a value that wasn't a clean multiple of 20 (the number of play-grid rows), so the renderer's cell size (`canvas.width / 10`) had a fractional residual and 20×cell didn't equal the canvas height.

### Fixed

- **Grid-locked sizing.** `--board-h` is now always snapped down to a multiple of `ROWS` (= 20). Each cell is now an integer number of CSS pixels, and 20×cell lands precisely on the bottom edge of the canvas — at every zoom level, on every device.
- **Honest mobile measurements.** Vertical budget on mobile is measured directly from the rendered HUD + vpad heights via DOM rects, replacing the previous hardcoded `220px` reservation. Phones, tablets, and shrunken windows all converge on the largest grid-aligned board that fits.
- **No more independent mobile clamps.** Mobile no longer applies separate `min(width, 80vw)` and `min(height, 160vw)` to the canvas — those broke the strict 1:2 ratio when one axis clamped but the other didn't. JS picks the board height; CSS just inherits.
- **Renderer bitmap stays in lockstep.** After every `applyBoardSize()`, the renderer re-syncs its DPR-scaled bitmap on a double `requestAnimationFrame` so layout has fully flushed before `getBoundingClientRect()` is read.

### Changed

- `BOARD_MIN_H = 280` (was 360) so very tight viewports can still show 14 px cells.
- Resize / orientationchange listeners debounced via rAF.
- Service worker cache bumped to `tetris-v1.3.0`.
