# tetris v1.5.0 — mobile rebuild

**Mobile is finally good.** The whole touch experience has been rebuilt from scratch around gestures, with the board filling the entire phone screen edge-to-edge.

## What's new on mobile

- **Board fills the viewport.** No chrome, no padding, no rounded corners. Just the playfield, edge-to-edge.
- **Five classic mobile Tetris gestures:**
  - **tap** → rotate
  - **drag left / right** → move (one cell per 22 px)
  - **drag down** → soft drop (one row per 22 px)
  - **swipe up** → hard drop
  - **long-press** → hold
- **Floating top bar** with HOLD · score · level · lines · time · NEXT — backdrop-blurred over the board, doesn't displace it.
- **Discovery vpad.** The virtual D-pad is always present at 18 % opacity so new players see it exists. Any ambiguous touch reveals it at full opacity for 3 seconds. Tap a button to use it, tap the board to dismiss.

## Under the hood

- `availableBoardHeight()` now measures the floating top bar with `getBoundingClientRect()` instead of looping over `.stage` siblings — the bar is `position:fixed` and was being missed.
- `availableBoardWidth()` on mobile uses the full `window.innerWidth` — the board frame has zero padding and zero border, so the canvas gets every pixel.
- `bindTouch()` rewritten: long-press detection, swipe-up = hard drop, ambiguous-touch → vpad reveal, `touchcancel` cleanup. Long-press fires once and consumes the trailing touchend so it doesn't also rotate.
- `Renderer` mirrors HOLD and NEXT previews into the floating bar's mini-canvases.

## What did *not* change

- **Desktop is untouched.** Same edge-to-edge frame, same integer-cell sizing model, same right-edge fix from v1.4.3. If desktop was perfect for you in v1.4.3, it's still perfect in v1.5.0.
- The pixel-exact `cell` sizing model (the source of truth since v1.4.1) is preserved end-to-end.

## Service worker

Cache bumped to `tetris-v1.5.0`. First load after deploy refreshes the offline bundle automatically.
